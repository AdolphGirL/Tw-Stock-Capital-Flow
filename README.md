# 📊 台股資金流分析平台 — 技術架構與核心算法深度解析

本專案是一個專為台灣股市設計的量化分析平台，專注於「板塊輪動」與「產業資金動態」。透過 Clean Architecture 架構設計，將全台股 1,900+ 檔個股依據產業進行二級聚合，並透過多維度的量化引擎將大數據轉化為直觀的投資決策指標。

## 🏗️ 二、 系統架構與設計原則

平台嚴格遵循 **Clean Architecture (乾淨架構)**，實施「畫面負責渲染、導航負責動詞、基礎設施維持公共化」的解耦設計。

### 1. 核心分析引擎 (Core Engines)
系統的心臟由多個量化引擎組成，位於 `lib/domain/engines/`：
*   **資金流向引擎 (`CapitalFlowEngine`)**：結合成交量比率 (35%)、動量分數 (40%) 與延續力 (25%) 計算個股與板塊得分。
*   **生命週期引擎 (`LifecycleEngine`)**：利用斜率、加速度與穩定度，將產業劃分為「點火、主升、擴散、狂熱、出貨、退潮、死亡」七個階段。
*   **板塊輪動引擎 (`RotationEngine`)**：量化資金在產業間的流動軌跡，找出當前市場的流入 (To) 與流出 (From) 標的。
*   **市場情緒引擎 (`MarketSentimentEngine`)**：綜合漲跌家數、強勢產業佔比與熱錢強度，產出全域風控分數。

### 2. 導航中樞與依賴穿透
*   **CategoryNavigation**：升級為「全 App 靜態調度中心」，負責全螢幕頁面跳轉及 `CategoryHistoryRepository` 的跨頁面傳遞，確保數據鏈路完整。
*   **HomePage 降噪**：定位為「大盤診斷與核心指標」，將熱圖、卡片等功能分流至獨立一級分頁，提升加載速度與使用者體驗。

## 💾 三、 資料庫與抗災防護 (Database & Resilience)

底層採用 **SQLite (Drift)** 進行高效資料管理，路徑位於 `lib/data/database/`。

### 1. 資料表結構 (`category_history`)
紀錄包含：`tradeDate`、`categoryName`、`score` (資金流)、`hotScore` (熱度)、`persistence` (延續力) 等關鍵量化欄位。

### 2. 三大防禦線
*   **Bootstrap Cache**：開屏時進行秒級攔截，若已有今日數據則直接命中快取，拒絕重複下載。
*   **Background Sync**：今日數據在 Isolate 背景執行緒解析後，無感寫入硬碟。
*   **離線降級 (Offline Graceful Degradation)**：當網路異常或逾時，自動繞過日期限制，檢索最近一次健康的歷史快照數據，實現「0 白屏、0 閃退」。

## 🧪 四、 實務使用與驗證

*   **大盤診斷**：啟動後於首頁查看市場多空分數，判定當前交易環境。
*   **產業穿透**：進入子產業頁面，利用原地滑出的成份股抽屜查看個股排行。
*   **外部擊穿**：點擊個股自動喚起瀏覽器並精準對接 Yahoo 奇摩股市（自動判定上市 .TW / 上櫃 .TWO）。
