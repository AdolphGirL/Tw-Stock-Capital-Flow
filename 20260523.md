# Tw-Stock-Capital-Flow (台股資金流) 專案分析報告 - 2026/05/23

## 專案架構分析

### 1. 目錄結構診斷
目前專案採用了類 **Layered Architecture (分層架構)**，結合了 **Engine-based** 的運算邏輯分離：
- **`core/engines/`**: 這是專案的核心競爭力。將複雜的台股演算邏輯（資金流、週期、情緒）抽離成獨立的 Engine，有助於邏輯測試與複用。
- **`services/` & `repositories/`**: 負責資料的持久化 (Storage) 與 外部獲取 (StockService)，初步具備了資料存取層的雛形。
- **`ui/`**: Page 與 Widget 分離清晰，並使用了較現代的 `flutter_animate` 增加視覺質感。

### 2. 架構優缺點
- **優點**: 
    - 邏輯運算高度模組化。
    - 命名規範符合台股業務邏輯（Mainstream, Rotation, Lifecycle）。
    - UI 元件化程度高 (Widgets 目錄豐富)。
- **缺點**:
    - **狀態管理缺失**: 目前 `main.dart` 承載了過多的初始化邏輯與全域狀態管理 (setState)，隨著功能增加，`main.dart` 將變得難以維護。
    - **依賴耦合**: Engine 的實例化主要發生在 `main.dart` 或 `CapitalFlowAnalyzer` 中，缺乏 **Dependency Injection (DI)**，難以進行單元測試。

## 潛在問題與警告 (瓶頸與壞味道)

1.  **效能瓶頸 (Main Thread Blocking)**:
    - 在 `main.dart` 的 `initialize()` 中，所有 Engine 的運算都在同一個非同步流程中執行。如果資料量大，頻繁的 `double` 運算可能會造成 UI 幀率掉落。
    - `CapitalFlowAnalyzer` 在計算主類股時，會重複計算子類股與個股分數，存在大量的重複運算。
2.  **Prop Drilling (屬性傳遞過深)**:
    - 大量的分析結果從 `main.dart` 一路傳遞到 `HomePage` 以及各個子頁面，這會使得 UI 組件與資料模型高度耦合。
3.  **錯誤處理 (Robustness)**:
    - `SyncManager` 失敗時僅輸出 String 到 Error 變數，缺乏針對網路錯誤、磁碟空間不足或 API 格式變動的細緻化處理。
4.  **冗餘代碼與空文件**:
    - 專案中存在數個空文件，增加了專案的混亂度。

## 重構與優化建議

### 1. 建議使用的套件
- **狀態管理**: 建議引入 **Riverpod**。其 `FutureProvider` 非常適合處理目前的 `initialize()` 流程，並能自然地將 `Engines` 的運算結果快取。
- **依賴注入**: 使用 **Get_It**。將 `StorageService`, `StockService`, 以及各個 `Engines` 註冊為單例，避免重複實例化。
- **資料模型**: 使用 **Freezed**。目前的 Model 多為手寫 `fromJson/toJson`，且不可變性 (Immutability) 沒被嚴格執行。
- **並行運算**: 對於大數據量的 `Engine.analyze()`，建議使用 `Isolate.run()` 將運算移出 UI Thread。

### 2. 核心演算邏輯說明

| 演算模組 | 核心邏輯敘述 | 關鍵指標 |
| :--- | :--- | :--- |
| **資金流分數 (Capital Flow)** | 結合 **量增比 (Volume Ratio)**、**價格動能 (Momentum)** 與 **多頭持續性**。 | `(量比*0.35) + (動能*0.4) + (持續性*0.25)` |
| **主流類股 (Mainstream)** | 評估類股的集體性表現。結合資金流入強度、擴散度 (上漲家數比) 與領頭羊表現。 | `(資金流*0.35) + (持續性*0.3) + (擴散*0.2) + (領頭羊*0.15)` |
| **市場情緒 (Sentiment)** | 反映全市場的過熱或恐慌程度。綜合上漲家數比、強勢類股數量與熱錢總強度。 | 分數區間：<20 (恐慌) 到 >85 (狂熱) |
| **生命週期 (Lifecycle)** | 根據分數的 **斜率 (Slope)** 與 **加速度 (Acceleration)** 判斷所處階段（點火、主升、狂熱、出貨等）。 | 判斷指標：動能加速度、穩定度、波動率 |
| **資金輪動 (Rotation)** | 尋找分數衰退 (From) 與分數增加 (To) 的對應關係，捕捉資金流向的變化。 | `(流出強度 + 流入強度) / 2` |
